## 1. 项目定位

- 项目名：`WordSprint`
- 类型：轻量单词卡 / 记忆训练平台
- 架构：`Vue 3 + Spring Boot + Redis + MySQL`
- 目标：用于 `Java 学习`，不是商业化大系统
- 当前优先级：先完成 MVP，再做增强功能

禁止把本项目擅自扩展为：

- 微服务系统
- 多端统一平台
- 实时聊天平台
- AI 推荐系统
- 大而全教育平台

## 5. 技术约束

## 5.1 后端

- Java 版本使用 `Java 17`
- 框架使用 `Spring Boot 3.x`
- 数据访问优先使用 `MyBatis-Plus`
- 鉴权优先使用 `Spring Security + JWT`，除非用户明确改用 `Sa-Token`
- 数据库使用 `MySQL 8`
- 缓存使用 `Redis 7`

## 5.2 前端

- 使用 `Vue 3`
- 构建工具使用 `Vite`
- 状态管理使用 `Pinia`
- 路由使用 `Vue Router`
- 请求库使用 `Axios`
- UI 组件库优先 `Element Plus`

## 5.3 不允许随意引入

- 新状态管理方案
- 新 ORM
- 新鉴权体系
- CSS 大型方案替换
- 与当前项目规模不匹配的基础设施

## 7. 编码规范

### 后端规范

- `controller` 只处理请求和返回，不堆业务逻辑
- `service` 承担业务逻辑
- `mapper` 只负责数据访问
- `entity` 不直接返回前端
- 输入使用 `DTO`
- 输出使用 `VO`
- 统一返回结构
- 统一异常处理
- 参数校验优先使用注解校验

### 前端规范

- 页面级组件放 `views`
- 可复用组件放 `components`
- 所有接口统一走 `api` 层
- 不在页面中散写裸 `axios`
- 路由守卫统一处理登录态
- Token 管理统一封装

### 通用规范

- 优先可读性，不为“高级感”增加理解成本
- 避免复制粘贴，但也不要过度抽象
- 方法命名要直接表达业务意图
- 不写与当前问题无关的“顺手优化”

## 8. 数据库约束

- 表名、字段名统一使用 `snake_case`
- 主键统一为 `id`
- 时间字段统一使用 `created_at`、`updated_at`
- 删除优先逻辑删除
- 高频查询要建索引
- 状态字段使用明确枚举值，不写魔法数字

若修改数据库设计，必须同步更新：

- `DATABASE_DESIGN.md`
- 相关 SQL 文件
- 涉及的接口文档

---

## 9. Redis 约束

Redis 在本项目中只能承担以下职责：

- 排行榜
- 热点缓存
- 随机抽题池
- 签到状态
- 幂等锁 / 防重复提交

必须遵守：

- Key 使用统一前缀：`wordsprint:`
- 临时数据必须设置 TTL
- 核心数据不能只保存在 Redis
- 更新数据库后要考虑缓存失效或刷新

禁止：

- 把 Redis 当主数据库使用
- 无限制堆积永久 key
- 未定义失效策略就直接缓存对象

---

## 10. API 约束

- 基础前缀统一为 `/api`
- 风格以 REST 为主
- GET 查询，POST 新增，PUT 更新，DELETE 删除
- 返回结构统一
- 所有受保护接口默认需要登录
- 涉及用户私有数据时必须校验归属权

如果新增或修改接口，必须同步更新：

- `API_LIST.md`
- 相关 DTO / VO
- 前端对应 `api` 封装
